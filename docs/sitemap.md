# Sitemap Automation

## Architecture

`/sitemap.xml` is a Sitemap index. It points to type-specific UTF-8 XML files below `/sitemaps/`:

- `sitemap-pages.xml`
- `sitemap-categories.xml`
- `sitemap-products.xml`
- `sitemap-posts.xml`

Each file is automatically split before 49,000 URLs or 49 MB, leaving a safety margin below Google's 50,000 URL and 50 MB limits. Only public, published, self-canonical URLs without query strings, `noindex`, robots blocks, admin/API paths or search-result paths are accepted.

Every URL has a content fingerprint. An unchanged fingerprint keeps its prior `lastmod`; a significant content change records the detected update time. Published CMS timestamps are used on first discovery. Removed, draft or offline content is removed at the next generation.

Production snapshots, URL state and run logs are stored in PostgreSQL tables using `CMS_TABLE_PREFIX`. Generation holds a PostgreSQL advisory lock and commits URL state plus validated XML snapshots in one transaction. A failed transaction leaves the previous working snapshot intact. Local development uses `.data/sitemap`, a stale-safe lock file, temporary writes and atomic rename.

## Triggers

- Product/news create, update, status change, slug change and delete queue a non-blocking refresh.
- News automation queues a refresh after successful publication.
- Vercel runs `/api/cron/sitemap?submit=true` daily at `09:23 UTC`.
- `/admin/sitemap` provides authenticated check, force-generate and generate-plus-submit controls.

Do not create a second operating-system cron for the production Vercel deployment.

## Environment

```env
SITE_URL=https://cowinmagnet.cl
SITE_CONTENT_UPDATED_AT=2026-07-10T00:00:00.000Z
CRON_SECRET=
DATABASE_URL=
CMS_TABLE_PREFIX=cowinmagnet_cl
GOOGLE_SEARCH_CONSOLE_ENABLED=false
GOOGLE_SEARCH_CONSOLE_SITE_URL=sc-domain:cowinmagnet.cl
GOOGLE_SEARCH_CONSOLE_SITEMAP_URL=https://cowinmagnet.cl/sitemap.xml
GOOGLE_SERVICE_ACCOUNT_BASE64=
```

The integration also accepts `GOOGLE_SERVICE_ACCOUNT_JSON`, `GOOGLE_SERVICE_ACCOUNT_CREDENTIALS_PATH`, or `GOOGLE_CLIENT_EMAIL` plus `GOOGLE_PRIVATE_KEY`. Never commit credentials.

## Search Console Authorization

1. Enable the Google Search Console API in the Google Cloud project.
2. Create a service account and obtain its email address.
3. Add that email as an authorized user for the `cowinmagnet.cl` Search Console property with permission to submit Sitemaps.
4. Store the credential securely in Vercel, preferably as `GOOGLE_SERVICE_ACCOUNT_BASE64`.
5. Set `GOOGLE_SEARCH_CONSOLE_ENABLED=true` only after authorization.
6. Run a forced submission and confirm the API result in `/admin/sitemap`.

The implementation uses `PUT /webmasters/v3/sites/{siteUrl}/sitemaps/{feedpath}` with the `webmasters` OAuth scope. It does not call `google.com/ping` and does not use the Indexing API for ordinary pages.

## Manual Commands

Load the deployment environment so `CRON_SECRET` is available, then run:

```bash
npm run sitemap:generate -- --dry-run --verbose
npm run sitemap:generate -- --force
npm run sitemap:generate -- --force --submit --verbose
```

- `--force`: regenerate even when no content fingerprint changed.
- `--dry-run`: compare without persisting snapshots or logs.
- `--submit`: submit the resulting index through Search Console API.
- `--verbose`: print the sanitized run result.

## Logs

Open `/admin/sitemap` to see start/end time, duration, trigger, files, URL counts, added/modified/deleted URLs, bytes, split state and Search Console result. Runtime logs use the `[sitemap]` prefix and never include tokens or credentials.

## Tests

```bash
npm test
npm run typecheck
npm run build
```

Tests cover XML generation and escaping, exclusion rules, deletion, stable `lastmod`, splitting, Sitemap index generation, locking, atomic write failure, successful Search Console submission and authorization failure isolation.

## Troubleshooting

- **Sitemap 404/503:** check `/admin/sitemap`, `DATABASE_URL`, deployment function logs and the latest failed run.
- **Invalid XML:** run `npm test`, then force generation. The previous valid snapshot remains available when validation fails.
- **robots.txt missing Sitemap:** verify `https://cowinmagnet.cl/robots.txt`; it is generated by `app/robots.ts`.
- **Search Console 403:** confirm the service-account email is authorized on the exact `sc-domain:cowinmagnet.cl` property and that the Search Console API is enabled.
- **Submitted but not indexed:** inspect the URL in Search Console, confirm HTTP 200, self-canonical metadata, crawl permission, useful content and internal links.

Successful Sitemap submission does not mean Google has crawled it. Crawling does not guarantee indexing. A Sitemap mainly helps Google discover canonical URLs; final indexing status must be confirmed in Google Search Console.
